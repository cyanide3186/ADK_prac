# **🤖 AI 에이전트 설계 문서 초안: CI/CD 자율 운영 코디네이터**

## **1\. 개요 및 목표**

### **1.1 에이전트 목표 정의**

본 에이전트('CI/CD 자율 운영 코디네이터')는 형상관리자(CM)의 **경험치 의존적인 빌드/테스트 트러블 슈팅, 이슈 전파, 소스코드 마이그레이션 관리** 등의 핵심 업무를 대리하고 자동화합니다.

* **최종 목표:** CM의 손을 타는 업무를 최소화하고, 이슈 발생 시 **전반적인 피드백 속도를 획기적으로 향상**하여 고품질의 개발 문화 구축에 기여합니다.

### **1.2 핵심 능력: 가치 판단 및 자율성**

단순한 작업 수행을 넘어, 다음 두 가지 핵심 영역에서 **고차원적인 가치 판단**을 수행하도록 설계됩니다.

1. **우선순위/경중 판단:** 이슈 레포트 생성 시 경중(Severity)을 자동으로 부여하고, 담당자 지정의 우선순위를 결정합니다.  
2. **심층 원인 및 책임자 판단:** 로그 분석을 통해 **CM의 경험치**를 시뮬레이션하여 근본 원인 코드를 색출하고, Git 기록을 바탕으로 **최적의 책임/담당자**를 지정합니다.

---

## **2\. 에이전트 아키텍처 (멀티 에이전트 시스템)**

본 에이전트는 효율성과 확장성을 위해 \*\*Supervisor Agent(관리자) 아래 두 개의 Sub Agent(전문가)\*\*를 두는 멀티 에이전트 시스템으로 설계됩니다.

### **2.1 EMR 구조 요약**

| 구성 요소 | 역할 (Entity) | 관계 (Relationship) | 책임 (Method/Responsibility) |
| :---- | :---- | :---- | :---- |
| **Supervisor Agent** (최상위) | **CI/CD 자율 운영 코디네이터** | **Delegate** (위임) | **상황 인지, 태스크 분류 및 위임** (어떤 Sub Agent에게 넘길지 결정), **결과 취합 및 최종 보고.** |
| **Sub Agent 1** | **빌드/테스트 이슈 분석 에이전트** | **Report** (보고) | **로그 확보 및 심층 분석, 원인 코드 색출, 담당자 지정, 이슈 전파.** |
| **Sub Agent 2** | **마이그레이션 관리 에이전트** | **Report** (보고) | **코드 충돌 분석, 마이그레이션 트러블 슈팅 및 품질 검증, 최종 레포트 작성.** |

## **3\. 에이전트의 워크플로우 및 사양**

### **3.1 Supervisor Agent의 지침 (Instruction)**

"나는 CI Unit의 목표 달성을 위해 CI/CD 파이프라인 전체를 모니터링하고, 발생하는 모든 빌드/테스트 및 마이그레이션 이슈를 가장 빠르고 정확하게 처리하는 코디네이터이다. 상황을 정확히 인지하여 두 전문 Sub Agent 중 한 명에게 태스크를 위임하고, 결과를 취합하여 최종적으로 사용자에게 보고한다."

### **3.2 필수 도구 (Required Tools)**

| 도구 그룹 | 도구명 (함수명) | 기능 |
| :---- | :---- | :---- |
| **데이터 접근/RAG** | read\_build\_log(source) | Jenkins/CI Tool에서 빌드 및 테스트 로그를 가져옴. |
|  | query\_artifact\_info(id) | 빌드 산출물의 버전 및 메타 정보에 접근. |
| **형상/버전 관리** | find\_last\_modifier(file\_path) | Git/Gerrit에서 특정 파일의 최근 수정자 이메일/ID를 검색. |
|  | analyze\_code\_conflict(files) | 마이그레이션 충돌 상세 분석. |
| **액션/보고** | create\_issue\_report(details, severity) | Jira 등 Issue Tracking System에 이슈 레포트 생성 및 **경중(Severity) 자동 부여.** |
|  | assign\_task(issue\_id, user\_id) | 이슈 트래킹 시스템에서 태스크를 할당. |
|  | send\_alert\_notification(user\_id, message) | 이메일, Slack 등을 통한 즉각적인 알림 발송. |

### **3.3 핵심 자율 워크플로우 정의**

#### **📝 워크플로우 A: 빌드/테스트 이슈 자율 트러블 슈팅 (Sub Agent 1\)**

1. **이벤트 감지:** Supervisor로부터 빌드 실패 이벤트 위임.  
2. **로그 확보 및 분석:** read\_build\_log 호출 후, 확보된 로그를 CM 경험치 기반으로 분석하여 \*\*오류 유형(코드/환경)\*\*과 **원인 코드 파일**을 도출.  
3. **책임자 지정:** find\_last\_modifier를 호출하여 **원인 코드 파일의 최종 수정자**를 책임자로 판단. (환경 문제 시 CM에게 할당).  
4. **전파:** create\_issue\_report를 사용하여 상세 리포트 작성 및 경중(Severity) 판단, assign\_task로 책임자에게 자동 할당, send\_alert\_notification으로 즉시 통보.

#### **📝 워크플로우 B: 소스코드 마이그레이션 관리 (Sub Agent 2\)**

1. **이벤트 감지:** Supervisor로부터 마이그레이션 충돌 또는 관련 빌드 오류 위임.  
2. **충돌 분석:** analyze\_code\_conflict를 호출하여 충돌 지점을 분석하고 필요한 해결 정보를 정리하여 중간 보고서 초안 작성.  
3. **트러블 슈팅:** 마이그레이션 과정에서 발생하는 빌드/테스트 이슈를 1차로 분석하고 해결 방안을 제시(트러블 슈팅).  
4. **최종 보고:** 마이그레이션 완료 후, query\_artifact\_info 등을 종합하여 마이그레이션의 품질을 판단하고 **전체 마이그레이션 레포트를 자동 작성**하여 Supervisor에게 보고.
